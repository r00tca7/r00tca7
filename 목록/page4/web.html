웹 기본 개념

1. 클라이언트 (Client)

정의: 클라이언트는 웹 애플리케이션이나 웹사이트를 사용하는 사용자 측의 장치나 소프트웨어를 말합니다. 일반적으로 웹 브라우저(Chrome, Firefox 등)가 클라이언트 역할을 합니다.

역할: 사용자가 웹 서버에 요청을 보내고, 서버로부터 응답을 받아 사용자에게 보여주는 역할을 합니다.

2. 서버 (Server)

정의: 서버는 클라이언트의 요청을 처리하고, 필요한 데이터를 제공하는 컴퓨터나 소프트웨어를 의미합니다.

역할: 클라이언트가 요청한 정보를 저장하고, 요청에 따라 HTML, CSS, JavaScript 파일 등을 클라이언트에게 전송합니다.

3. HTTP (HyperText Transfer Protocol)

정의: HTTP는 클라이언트와 서버 간의 통신 규약입니다.

역할: 클라이언트가 서버에게 요청을 보내고, 서버가 클라이언트에게 응답을 보내는 방식을 정의합니다.

4. URL (Uniform Resource Locator)

정의: URL은 웹 자원의 주소를 나타내는 문자열입니다.

역할: 클라이언트가 요청할 자원의 위치를 지정합니다.

5. HTML (HyperText Markup Language)

정의: HTML은 웹 페이지의 구조를 정의하는 마크업 언어입니다.

역할: 웹 브라우저가 내용을 표시할 수 있도록 구조화된 텍스트, 이미지, 비디오 등의 자원을 정의합니다.

6. CSS (Cascading Style Sheets)

정의: CSS는 웹 페이지의 스타일을 정의하는 언어입니다.

역할: HTML 요소의 색상, 글꼴, 레이아웃 등을 설정하여 웹 페이지의 시각적 표현을 담당합니다.

7. JavaScript

정의: JavaScript는 웹 페이지에 동적인 기능을 추가하는 프로그래밍 언어입니다.

역할: 사용자와의 상호작용을 처리하고, 웹 페이지의 내용이나 스타일을 동적으로 변경할 수 있게 해줍니다.

8. 데이터베이스 (Database)

정의: 데이터베이스는 데이터를 구조화하여 저장하는 시스템입니다.

역할: 서버가 클라이언트의 요청에 따라 필요한 데이터를 저장하고 검색하는 데 사용됩니다.

9. API (Application Programming Interface)

정의: API는 서로 다른 소프트웨어 시스템 간의 상호작용을 가능하게 하는 인터페이스입니다.

역할: 클라이언트가 서버와 통신하거나, 다른 서비스와 데이터를 주고받기 위해 사용됩니다.
